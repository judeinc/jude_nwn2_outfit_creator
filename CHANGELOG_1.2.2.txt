Version 1.2.2 Work Log / Changelog

Summary

- Adds profile-specific Step 4 shaping presets for OOM and OOF.
- Keeps the existing modular body-profile approach so future races can be added as profile entries rather than one-off workflow branches.

Step 4 Profile Shaping
- Added a generic profile-shape preset registry.
- Added a new Step 4 operator:
  - nwn2.step4_profile_shape
- The Shape button appears only when the active profile has a preset.
- Current shape presets:
  - Shape to OOM
  - Shape to OOF
- HHM and HHF remain unchanged and do not show a profile-shape button.

OOM Shape Preset
- Added OOM upper-body shaping from manual pose-mode fitting data.
- OOM preset adjusts torso, ribcage, shoulders/collar bones, arms, neck, and upper legs.
- OOM preset applies a forward shift target of 0.030.
- OOM preset includes a conservative leg bulk pass:
  - LLeg1 / RLeg1 scale: 1.18, 1.0, 1.18
  - LLeg2 / RLeg2 scale: 1.12, 1.0, 1.12
- OOM does not hard-code object Z placement; vertical placement remains controlled by Step 4 floor alignment and manual Shift Up/Down controls.

OOF Shape Preset
- Added OOF upper-body shaping from manual pose-mode fitting data.
- OOF preset adjusts torso, ribcage, shoulders/collar bones, arms, and upper legs.
- OOF preset applies a forward shift target of 0.010.
- OOF preset applies a small tilt target of -0.5 degrees.
- OOF preset includes a conservative leg bulk pass based on OOF body measurements:
  - LLeg1 / RLeg1 scale: 1.22, 1.0, 1.22
  - LLeg2 / RLeg2 scale: 1.14, 1.0, 1.14

Step 4 Fitting Controls
- Added Vertical Shift controls:
  - Shift Down
  - Shift Up
- Vertical Shift moves Ai_Rig on Blender Z in 0.01 increments.
- If newModel_Mesh is not parented to Ai_Rig, the mesh is moved along with the rig as a fallback.
- Vertical Shift resets when Restore to NWN2 Scale is run again.

Export / Late Workflow Verification
- Confirmed Substance Painter handoff is unaffected by body profile shaping.
- Confirmed Step 8 cleanup stamps the mesh profile and removes temporary profile assets generically.
- Confirmed Send to Export uses the stored profile skeleton.
- Confirmed MDB export selects the export mesh, COLS, and the matching profile skeleton only.
- Confirmed other profile skeletons are hidden/excluded during MDB export.
- Added a small MDB export cleanup guard so COLS and the active profile skeleton are re-hidden if export cancels because no export folder is selected.

Post-1.2.2 Restore Reference Fix
- Fixed a repeated-session restore bug after sending an outfit to the Repository and starting a new outfit.
- Restore References for Neverwinter Nights now safely relinks starter/reference assets into NWN2_Working before applying visibility.
- Added duplicate starter-object normalization for Blender-suffixed names such as P_HHF_skel.001.
- Visibility restore now uses a safe hide_set wrapper so objects outside the active ViewLayer cannot crash the restore process.
- Duplicate starter rig cleanup now retargets children, Armature modifiers, and constraints to the clean base rig before removing the duplicate.
- Restore now repairs starter body/boots/gloves/head/mannequin parenting if a previous restore orphaned them from their profile skeleton.
- Corrected Repository safety so finished game-style outfit names such as P_HHM_CL_Body130 and P_HHM_CL_Body131 are never canonicalized into starter references.
- Step 1 Identify AI Mesh now ignores meshes stored in NWN2_Repository and NWN2_Export while still detecting the newly imported working mesh.
- Starter duplicate normalization is limited to true Blender duplicate names such as P_HHF_skel.001 and P_HHM_CL_Body01.001, not valid numbered outfit variants.
- Verified manually in Blender after the user reproduced and confirmed the fix.

Post-1.2.2 Batch Export Addition
- Added Batch Export Repository under Step 8.
- Batch export shows a warning prompt before folder selection explaining that processing can take time and Blender may appear busy.
- Batch export runs a preflight before exporting any MDB files.
- Preflight checks for Repository meshes, COLS, matching profile skeletons, duplicate export names, mesh data, and vertex groups.
- Texture preflight warns about missing _D / _N images and missing texconv.exe when texture export is enabled.
- Batch export creates one folder per repository outfit inside the chosen export folder.
- Batch export reuses the existing single MDB export path for each outfit so naming, DDS export, COLS selection, profile skeleton selection, and node/image cleanup stay consistent.
- Batch export stops on the first export error instead of continuing through the remaining outfits.
- Batch export restores the Export slot and keeps Repository outfits in the Repository after processing.
